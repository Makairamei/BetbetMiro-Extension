# Code of Conduct

## Purpose

BetbetMiro Extension is maintained to keep CloudStream provider development organized, reviewable, and useful for users.

This Code of Conduct sets basic expectations for issues, pull requests, discussions, and any repository-related collaboration.

---

## Expected Behavior

Participants are expected to:

- Be respectful and constructive.
- Keep reports and pull requests focused on the technical issue.
- Provide clear evidence when reporting provider breakage.
- Respect maintainer time.
- Avoid spam, repeated demands, or low-effort reports.
- Follow GitHub rules and applicable laws.
- Keep adult/NSFW-related discussion strictly technical and repository-relevant.

Healthy disagreement is allowed. Personal attacks are not.

---

## Unacceptable Behavior

The following behavior is not acceptable:

- Harassment, insults, threats, or personal attacks.
- Hate speech or discriminatory language.
- Posting illegal, abusive, or explicit media.
- Sharing private information without permission.
- Demanding immediate fixes without reproduction details.
- Opening repeated duplicate issues after being asked to provide evidence.
- Submitting broad unrelated changes disguised as provider fixes.
- Using issues or pull requests to promote piracy, bypass paywalls, or request illegal access.

---

## Provider Issue Conduct

When reporting a provider issue, keep the report factual.

Good reports include:

- Provider name.
- Problematic URL.
- Steps to reproduce.
- CloudStream version.
- Screenshot or log when useful.
- Whether homepage, detail, or playback is broken.

Bad reports include:

- "Not working" with no context.
- Demanding fixes without evidence.
- Posting explicit screenshots when text/logs would be enough.
- Mixing multiple unrelated providers in one issue.

---

## Pull Request Conduct

Pull Requests should be reviewable and focused.

A good PR:

- Explains the root cause.
- Describes the fix.
- Provides source evidence when changing provider parsers/resolvers.
- Bumps provider version when provider code changes.
- Avoids unrelated file changes.
- Clearly states build and runtime validation status.

A PR may be closed or requested for changes if it is too broad, unsupported by evidence, or risky for provider stability.

---

## Maintainer Enforcement

Maintainers may take action to keep the repository healthy, including:

- Asking for clarification or reproduction evidence.
- Editing labels or issue titles for clarity.
- Closing incomplete or duplicate issues.
- Requesting changes on Pull Requests.
- Locking conversations that become unproductive.
- Blocking users who repeatedly violate this Code of Conduct.

Enforcement should be proportional to the behavior and focused on keeping the project maintainable.

---

## Reporting Problems

If you see behavior that violates this Code of Conduct, report it through GitHub using the most appropriate channel.

For normal provider bugs, use the issue templates.

For sensitive concerns, contact the repository maintainer through the available GitHub profile or repository contact channel.

---

## Scope

This Code of Conduct applies to:

- Issues.
- Pull Requests.
- Review comments.
- Discussions connected to this repository.
- Any collaboration directly related to BetbetMiro Extension.

The goal is simple: keep the project useful, respectful, and maintainable.
