## Contributors

Contributors may be credited through:

- Git commit history.
- Pull Request authorship.
- Issue reports with useful source evidence.
- Changelog entries when a contribution materially affects provider behavior.

Good contribution credit should be factual and verifiable.

---

## Source Websites

Provider source websites, logos, names, catalogs, streams, subtitles, and metadata belong to their respective owners.

This repository does not claim ownership of third-party source content.

Provider code should be maintained based on active source behavior, source evidence, and CloudStream app behavior.

---

## Upstream / Reference Evidence

When provider flow is aligned with verified upstream/source behavior, use professional wording such as:

- `update provider flow from upstream source evidence`
- `align parser with active source behavior`
- `resolver adjusted from source evidence`
- `Credit: <verifiable project/user/source>`

Avoid unclear wording that makes review harder.

Do not credit unverified references.

---

## Issue Report Credits

Useful reports may be credited when they include:

- Provider name.
- Working or broken source URL.
- Reproduction steps.
- Screenshot/log/HAR evidence when appropriate.
- Clear note whether homepage, detail, or playback is affected.

Low-context reports such as `not working` are not enough for technical credit.

---

## Pull Request Credits

Pull Request authors should include enough context for review:

- Root cause.
- Exact fix.
- Changed files.
- Provider version bump.
- Build status.
- Runtime validation status.

Credit does not replace validation. A credited change must still follow repository maintenance standards.
